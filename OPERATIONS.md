# JXYD 非工程人員操作手冊（維運版）

> 目的：讓不懂程式的人，也可以安全操作 JXYD 並留下維護紀錄。

## 0. 你會用到的三個入口
- 網站前台：一般訪客看到的頁面。
- 管理模式（⚙）：新增/刪除文章與影片。
- 瀏覽器主控台（Console）：只在需要改管理密碼時使用。

---

## 1. 每次維運前的「3 分鐘安全檢查」
1. 打開網站，確認首頁可正常載入。
2. 點右上角 ⚙，確認管理密碼視窗可打開。
3. 隨機點一個外部連結，確認會開新分頁。

若以上任一步失敗，先暫停改動，記錄狀況並通知維護人員。

---

## 2. 日常操作（新增/刪除內容）

### 2.1 新增文章
1. 點 ⚙ 進入管理模式。
2. 切到「新增文章」。
3. 填入：日期、標題、標籤、連結、狀態。
4. 按「新增」。
5. 回前台確認文章已出現。

### 2.2 新增影片
1. 點 ⚙ 進入管理模式。
2. 切到「新增影片」。
3. 填入：日期、標題、觀看數（可空）、連結、狀態。
4. 按「新增」。
5. 回前台確認影片已出現。

### 2.3 刪除與清空
- 單筆刪除：在「管理內容」中按對應「刪除」。
- 全部清空：按「清空全部」，系統會再確認一次。

> 注意：清空會移除本機瀏覽器資料，操作前請先確認。

---

## 3. 修改管理密碼（不改網頁外觀）

### 3.1 正確做法（請完整複製，使用半形引號）
1. 開啟網站，按 `F12` 進入「主控台 Console」。
2. 輸入以下指令（請用 **半形單引號** `'`，不要用中文引號 `‘ ’`）：

```js
setAdminPassword('MyNewPassw0rd!2026')
```

3. 若回傳 `true` 表示設定成功。
4. 重新按 ⚙ 測試新密碼能登入。

### 3.2 回復預設密碼
```js
resetAdminPassword()
```

### 3.3 檢查目前密碼來源（僅供確認）
```js
getAdminPassword()
```

## 4. 你截圖中兩個錯誤代表什麼

### 錯誤 A：`setAdminPassword is not defined`
代表目前頁面腳本沒有暴露這個函式（常見原因：
- 你開的是舊版頁面/快取頁。
- 部署版本尚未更新。
- 尚未載入最新版腳本）。

**處理方式**
1. 先強制重整（Windows: `Ctrl+F5` / Mac: `Cmd+Shift+R`）。
2. 再輸入 `typeof setAdminPassword`，應顯示 `"function"`。
3. 若仍不是 function，請確認部署版本是否包含最新 commit。

### 錯誤 B：`Invalid or unexpected token`
代表輸入指令有非法字元，最常見是用了中文引號或把密碼打到引號外。

**正確範例**
```js
setAdminPassword('1@345trewqAz')
```

## 5. 維護記錄怎麼留（非常重要）
每次操作完，請記在你的維運紀錄（例如 Notion / Google Sheet / issue）至少包含：
- 日期時間
- 操作人
- 做了什麼（新增幾篇、刪了什麼、是否改密碼）
- 驗證結果（成功/失敗）
- 異常與處理

建議模板：

```text
[日期時間]
[操作人]
[操作] 新增文章 2 篇；刪除影片 1 筆；密碼更新（是/否）
[驗證] 前台顯示正常；管理模式可登入
[備註] 無
```

---

## 6. 發生問題時的 SOP
1. **先停手**，不要連續嘗試清空或重複刪除。
2. 截圖錯誤訊息（含 Console）。
3. 記錄「你剛剛按了什麼」。
4. 通知維護人員，附上截圖與時間。

---
